Preguntas de la actividad

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la red mundial de redes interconectadas que permite la comunicación entre computadoras a través de protocolos como TCP/IP. La World Wide Web (WWW) es un servicio que funciona sobre Internet, basado en documentos de hipertexto vinculados entre sí mediante URLs y accesibles a través del protocolo HTTP/HTTPS. Es decir, Internet es la infraestructura y la Web es uno de los muchos servicios que utilizan esa infraestructura (otros servicios son el correo electrónico, FTP, etc.).

Fuente: MDN Web Docs

2. ¿Cuáles son las partes de un URL?

Las partes principales de un URL son:

  • Protocolo (esquema): ej. https://
  • Dominio (host): ej. www.ejemplo.com
  • Puerto: ej. :443 (opcional)
  • Ruta (path): ej. /ruta/recurso
  • Parámetros (query string): ej. ?id=123&cat=4
  • Fragmento (anchor): ej. #seccion

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: obtiene un recurso del servidor.
  • HEAD: obtiene solo los encabezados de la respuesta, sin el cuerpo. Útil para verificar si un recurso existe.
  • POST: envía datos al servidor para crear un nuevo recurso.
  • PUT: reemplaza por completo un recurso existente.
  • PATCH: modifica parcialmente un recurso existente.
  • DELETE: elimina un recurso del servidor.

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña? ¿Por qué?

Se debe utilizar el método POST, porque envía los datos en el cuerpo de la petición y no en la URL. Esto evita que información sensible como contraseñas quede expuesta en el historial del navegador, en los registros del servidor o en los marcadores. Además, POST permite enviar mayor cantidad de información que GET.

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET, ya que el navegador solicita al servidor el recurso (la página HTML) que corresponde al URL ingresado.

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa? ¿Ocurrió algún error?

El código 200 OK indica que la petición fue procesada con éxito. No ocurrió ningún error; la respuesta del servidor contiene el recurso solicitado.

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta un error 404? ¿Por qué?

Depende. El error 404 (Not Found) significa que el recurso solicitado no existe en el servidor. Si el usuario llegó al 404 porque siguió un enlace roto dentro del propio sitio o porque el desarrollador eliminó/movió una página sin redirigir, sí es responsabilidad del desarrollador corregirlo. Sin embargo, si el usuario simplemente escribió mal la URL, no es responsabilidad del desarrollador, aunque sí es buena práctica ofrecer una página 404 personalizada que ayude a navegar al sitio correcto.

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta un error 500? ¿Por qué?

Sí. El error 500 (Internal Server Error) indica que ocurrió un fallo en el lado del servidor, lo cual generalmente es responsabilidad del desarrollador o del equipo de infraestructura. Puede deberse a errores en el código, fallos de configuración, problemas con la base de datos, etc. El usuario no puede solucionarlo desde su lado.

9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado? Menciona algunos elementos de HTML 4 desaprobados en HTML5.

Un elemento o atributo depreciado (deprecated) es aquel que aún funciona pero cuyo uso ya no se recomienda, porque existen mejores alternativas (generalmente CSS) o porque será eliminado en futuras versiones del estándar.

Algunos elementos de HTML 4 desaprobados en HTML5 son:

  • <font> — reemplazado por CSS
  • <center> — reemplazado por CSS (text-align)
  • <big> — reemplazado por CSS
  • <strike> — reemplazado por <s> o <del>
  • <tt> — reemplazado por CSS o <code>
  • <frame> y <frameset>
  • <applet> — reemplazado por <object>

10. ¿Qué componentes de estructura y estilo tiene una tabla?

Componentes de estructura:

  • <table> — contenedor de la tabla
  • <caption> — título de la tabla
  • <thead> — encabezado
  • <tbody> — cuerpo
  • <tfoot> — pie
  • <tr> — fila
  • <th> — celda de encabezado
  • <td> — celda de datos
  • <colgroup> y <col> — agrupación de columnas

El estilo se aplica mediante CSS (bordes, colores, alineación, espaciado, etc.). Atributos como border, cellpadding y cellspacing existían en HTML 4 pero están desaprobados en HTML5.

11. ¿Cuáles son los principales controles de una forma HTML5?

  • <input> con tipos: text, password, email, number, date, tel, url, color, range, search, file, hidden, checkbox, radio, submit, reset, button
  • <textarea> — área de texto multilínea
  • <select> con <option> — listas desplegables
  • <button> — botones
  • <label> — etiquetas asociadas a controles
  • <fieldset> y <legend> — agrupación de controles
  • <datalist> — sugerencias para inputs
  • <output> — resultado de cálculos

12. Sobre el ciclo de vida y desarrollo de los sistemas de información

Ciclo de vida de un sistema de información: abarca todas las etapas desde que se concibe el sistema hasta que se retira. Generalmente incluye: planeación, análisis, diseño, desarrollo (implementación), pruebas, implantación, operación y mantenimiento, y finalmente el retiro o reemplazo del sistema.

Ciclo de desarrollo de sistemas de información (SDLC): es la parte del ciclo de vida enfocada específicamente en la creación del sistema. Sus fases típicas son:

  1. Análisis de requerimientos: entender qué necesita el usuario.
  2. Diseño: planear cómo se construirá el sistema.
  3. Implementación (codificación): escribir el código.
  4. Pruebas: verificar que todo funcione correctamente.
  5. Despliegue: poner el sistema en producción.
  6. Mantenimiento: corregir errores y agregar mejoras.

La diferencia clave es que el ciclo de vida es más amplio (incluye el retiro del sistema), mientras que el ciclo de desarrollo se enfoca solo en la construcción y entrega.
